# Week Chef

## Задача
Сделать помощника, который возьмет на себя задачу планирования питания на неделю с учетом возможностей пользователя:
* блюда подбираются под цели (снижение/поддержание/набор веса) и ограничения (аллергии, лечебные показания)
* выбор рецептов происходит на основании временных возможностей пользователя: анализ календаря и выбор слотов для готовки
* формирования списка покупок с учетом продуктов, которые уже есть дома
* изменение плана, если у пользователя возникли трудности (временные или продуктовые)

## Для кого
1. **Занятые студенты/молодые специалисты**: хаотичный график, питание “как получится”, хочется быстрее и дешевле.
2. **Люди с ограничениями по питанию**: непереносимости/аллергии, религиозные ограничения, “без молочки/глютена”, контроль калорий/белка, хочется разнообразия и простоты
3. **Пользователи, которые пытаются экономить**: хотят уменьшить траты и избегать “редких” ингредиентов ради одного блюда.

## Боль сейчас
- Планировщики питания часто игнорируют **календарь**: рецепт “45 минут” предлагается на день, где у человека окно 20 минут → план не исполняется
- Список покупок делается вручную и не учитывает остатки → перепокупки, выброшенные продукты
- Ограничения конфликтуют или забываются (аллергены/“не люблю”) → пользователь теряет доверие
- При первом же сбое нет механизма **оперативного перепланирования**: приходится начинать заново
Основная цель: сделать помощника, которому можно доверить свое питание, чтобы переключить свое внимание на удовольствие от готовки и еды, а не от планирования и трат

## Что именно сделает PoC на демо
### Демонстрационный сценарий (end-to-end)

1. **Ввод профиля**:
    - цель (например, ~2000 ккал/день, больше белка),
    - ограничения (например, без арахиса, без молочки),
    - предпочтения (2–3 нелюбимых продукта),
    - оборудование (без духовки),
    - бюджет/лимиты (опционально),
    - сколько готовить и сколько порций.
2. **Подключение календаря (Google Calendar)**:
    - агент читает дела на неделю и находит окна (например, 30–40 минут в будни, 2 часа в выходные).
    - если пользователь не использует календарь, то может описать возможные слоты самостоятельно
3. **Ввод текущих продуктов**:
    - в виде списка (например, курица, рис, яйца, овощи, йогурт — но йогурт будет запрещен при “без молочки”)
    - дополнительно: VLM для считывания фото холодильника/чека
5. **Генерация недельного плана**:
    - n приемов пищи на день
    - для каждого блюда: название, время готовки, нутриенты, пометки по аллергенам, рецепт
    - распределение по временным слотам
6. **Список покупок**:
    - агрегированный по категориям,
    - берет во внимание то, что уже есть,
    - учитывает скоропортящиеся продукты
7. **Сбой и перепланирование**:
    - имитируем ситуацию: “во вторник добавилась встреча” или “не успел приготовить”,
    - агент делает перепланировку: перенос/замена блюда на быстрое, под ингридиенты

## Что НЕ делает PoC (out-of-scope)
1. **Не медицинский продукт**
- Не даёт медицинских рекомендаций, не лечит
- Не гарантирует безопасность при сложных диагнозах (диабет 1 типа, почечные ограничения и т.п.).

2. **Не “идеальная” нутрициология**
- Не обещаем точный подсчет БЖУ до грамма, если источник данных неполный.
- Не делаем персональные нормы по анализам/биомаркерам.

3. **Не полноценная покупка продуктов**
- PoC генерирует **список покупок** (и, максимум, черновик корзины/экспорт), но **не оформляет заказ и не платит**.
- Не решаем полноценно задачу динамической доступности и цен во всех магазинах

4. **Не мультипользовательская/семейная оптимизация**
- Один пользователь, без сложного “у всех разные предпочтения/аллергии” и без оптимизации на 4 человек
